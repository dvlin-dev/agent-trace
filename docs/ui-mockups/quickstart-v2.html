New 数据流图 — 新增三段式可视化流图，替代原有纯文字 chevron 箭头，直观展示「你的客户端 → Agent Trace → 上游 API」的关系

新增引导态设计要点：
1. 绿色 checkmark + 呼吸灯表示 listener 已启动，正在等待。
2. 两步引导卡片：设置环境变量 → 正常使用 agent，降低认知负担。
3. 当第一个请求到达后自动跳转到 workspace（无需用户操作）。
4. 提供 "Skip to workspace" 跳过按钮（for 高级用户 / 已配置过的用户）。

交互总结

Step 用户操作 系统行为 v1 对比
1. 选 Provider 点选卡片 → Continue 预填默认 URL + 端口 Logo 更新，配色对齐
2. 配置连接 确认/修改 upstream URL → Start 创建 profile + 启动 listener 新增三段式流图
3. 等待流量 复制 export → 终端执行 → 使用 agent 首个请求到达 → 自动进 workspace 新增：引导等待态
实施注意事项：
1. Step 3（等待态）需要监听 proxy 的首次请求事件，通过 IPC push 通知 renderer 自动跳转。
2. Logo SVG 已存在于 resources/icon.svg，renderer 可通过 <img> 引入或直接 inline。
3. 所有 violet-500 / purple-600 引用需替换为 accent-brand token。
4. Provider emoji（🟠🟢）→ 使用 profile-switcher.tsx 中已有的 AN/CX badge 模式。
